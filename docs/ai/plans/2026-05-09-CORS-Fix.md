# Plano de Correção: CORS e Conectividade BlueBot

Este plano visa resolver os erros de política CORS que impedem o Console e o App de se comunicarem com a API do BlueBot.

## Situação Atual
- O Navegador bloqueia requisições do console (`https://console.bluebotapp.com.br`) para a API (`https://api.bluebotapp.com.br`) porque o cabeçalho `Access-Control-Allow-Origin` retornado é inesperado ou conflituoso.
- Atualmente, tanto o Nginx (`nginx.conf`) quanto o FastAPI (`main.py`) estão tentando configurar CORS.
- O Nginx intercepta requisições `OPTIONS` (preflight) e retorna um valor fixo, impedindo o funcionamento dinâmico do FastAPI.

## Problemas Identificados
1. **Conflito de Configuração**: Duplicidade de cabeçalhos CORS entre Nginx e Backend.
2. **Hardcoding no Nginx**: O Nginx força a origem `https://console.bluebotapp.com.br` mesmo quando a requisição vem de `https://app.bluebotapp.com.br`.
3. **Bloqueio de Preflight**: O FastAPI não recebe as requisições `OPTIONS`, então seu `CORSMiddleware` nunca entra em ação para validar as origens permitidas.

## Solução Proposta: Centralização no Backend
A melhor prática em arquiteturas modernas é deixar o Framework (FastAPI) gerenciar o CORS, pois ele possui lógica nativa para lidar com múltiplas origens de forma dinâmica e segura.

### [MODIFY] [nginx.conf](file:///c:/Users/mathe/Downloads/BlueBot/nginx.conf)
- Remover todos os blocos `add_header 'Access-Control-Allow-Origin' ...`.
- Remover todos os blocos `if ($request_method = 'OPTIONS') { return 204; }`.
- Isso permitirá que todas as requisições cheguem ao FastAPI, que já está configurado com `CORSMiddleware`.

### [MODIFY] [api/app/main.py](file:///c:/Users/mathe/Downloads/BlueBot/api/app/main.py)
- Garantir que a lista `allowed_origins` inclua todas as variantes necessárias.
- Adicionar `https://www.bluebotapp.com.br` e `http://localhost:3000` (comum em dev) se necessário.

## Cronograma de Implementação
1. **Sessão 1**: Backup e Limpeza do `nginx.conf`.
2. **Sessão 2**: Ajuste fino no `api/app/main.py`.
3. **Sessão 3**: Validação via script de teste (simulando Origin headers).

## Riscos e Mitigação
- **Risco**: Algum cliente antigo depender de cabeçalhos inseridos especificamente pelo Nginx.
- **Mitigação**: O `CORSMiddleware` do FastAPI é mais completo e segue os padrões RFC, cobrindo todos os casos de uso atuais.

## Critérios de Sucesso
- Requisições de `console.bluebotapp.com.br` para `api.bluebotapp.com.br` retornam status 200 com os cabeçalhos CORS corretos.
- Requisições de `app.bluebotapp.com.br` para `api.bluebotapp.com.br` funcionam da mesma forma.
- Fim dos erros de "CORS policy" no console do navegador.
